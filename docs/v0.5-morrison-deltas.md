# v0.5 Morrison Regression — Running Deltas

Per the plan's inline regression discipline, after each new rule lands we run it against
`packs/vv40/examples/morrison/cou1/uofa-morrison-cou1.jsonld` and
`packs/vv40/examples/morrison/cou2/uofa-morrison-cou2.jsonld` and record the delta here.

Baseline (`v0.4.0-nafems` tag):
- Morrison COU1: **14 weakeners** (W-EP-01×1, W-EP-02×3, W-AL-01×3, W-AR-05×3, COMPOUND-01×3, COMPOUND-03×1)
- Morrison COU2: **6 weakeners** (W-EP-04×6)
- Cross-COU divergences: 7

## W-ON-02 — Unbounded Applicability (High)

| | Count before | Count after | Delta | New firings |
|---|---|---|---|---|
| Morrison COU1 | 14 | 16 | +2 | `cou1-cpb` (W-ON-02, High) + cascade: +1 COMPOUND-01 (High/Critical pairing) |
| Morrison COU2 | 6 | 7 | +1 | `cou2-vad` (W-ON-02, High). No COMPOUND cascade — COU2 has no Critical weakener. |

**Interpretation:** Expected. Morrison examples were authored under v0.3/v0.4 before
`hasApplicabilityConstraint` and `hasOperatingEnvelope` were semantically required for a
clean COU. W-ON-02 is documenting a real gap in the assurance package, which is exactly
what the rule is meant to surface. No fix to the example data — let the weakener fire.

**Compound recount:** COU1's COMPOUND-01 count shifts +1 (3 → 4) because the new
High-severity W-ON-02 pairs with an existing Critical weakener through COMPOUND-01's
Critical+High cascade rule. COU2 unaffected (no Critical weakener on COU2).

## W-AR-03 — Inference Method Mismatch (High)

| | Count before | Count after | Delta | New firings |
|---|---|---|---|---|
| Morrison COU1 | 16 (+W-ON-02) | 16 | 0 | none |
| Morrison COU2 | 7 (+W-ON-02) | 7 | 0 | none |

**Interpretation:** Neither Morrison COU uses `uofa:requiredVerificationMethod` or
`uofa:activityType`, so the W-AR-03 preconditions fail and the rule does not fire.
Verified via `grep -l "requiredVerificationMethod|activityType" packs/vv40/examples/morrison/`
→ no matches.

**Running totals after Week 1 rules 1–2:** COU1 = 16, COU2 = 7.

## W-AL-02 — Sensitivity Gap (Medium)

| | Count before | Count after | Delta | New firings |
|---|---|---|---|---|
| Morrison COU1 | 16 | 16 | 0 | none — all 3 ValidationResults lack UQ (W-AL-01 fires 3×), so W-AL-02 precondition (UQ present) never holds |
| Morrison COU2 | 7 | 8 | +1 | UofA root — all 3 ValidationResults declare UQ but no `hasSensitivityAnalysis` on the UofA |

**Interpretation:** W-AL-01 and W-AL-02 are mutually exclusive on a per-result basis — if UQ is absent, W-AL-01 fires and W-AL-02 cannot. COU2 is the clean example of the W-AL-02 trigger: UQ is declared but the drivers of that uncertainty are not documented via a SensitivityAnalysis. This is a real quality gap.

**Running totals after Week 1 rules 1–3:** COU1 = 16, COU2 = 8.

## W-EP-03 — Stale Input Data (High)

| | Count before | Count after | Delta | New firings |
|---|---|---|---|---|
| Morrison COU1 | 16 | 16 | 0 | none |
| Morrison COU2 | 8 | 8 | 0 | none |

**Interpretation:** Neither Morrison COU declares `uofa:modelRevisionDate` on the UofA
nor `uofa:dataVintage` on any Dataset — these are new v0.5 properties that the existing
v0.4-authored Morrison packages do not use. W-EP-03 cannot fire without both present,
so preconditions fail silently. Rule semantics will be validated via unit-test fixtures
in Phase 3 (positive/negative/boundary cases).

**Running totals after Week 1 complete (rules 1–4):** COU1 = 16, COU2 = 8.
