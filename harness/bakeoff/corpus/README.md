# Bakeoff corpus — hardened, conflicting-signal cells

Generated by `harness/bakeoff/build_corpus.py` (the single source of truth — edit
the cells there and re-run `python -m harness.bakeoff.build_corpus`). **24 cells**
(19 fire, 5 control), 18 archetypes across 14 domains and 9 surrogate
proto-taxonomy D-categories (Jakeman et al., arXiv:2502.15496).

## What makes a cell hard (enforced, not assumed)

Every cell is a **conflicting-signal** cell: the *obvious* feature points the
**wrong** way (the §5A dangerous-OK property), so the correct disposition is not
latchable from one feature. `build_corpus.hardness_violations()` rejects any cell
whose obvious-signal posture equals the gold posture (an easy cell), and a test
asserts the same — so an easy cell that would inflate a clear cannot slip in.
Controls invert it: the obvious feature looks alarming (→ over-action) but the
correct disposition is *accept*.

## Provenance — honest, and deliberately layered

- `gold_mechanism` — **paper- or standard-grounded** (real citation per cell:
  AirfRANS, FNO failure modes, deep-ensemble UQ, CarBench/DrivAerML, or ASME
  V&V 40 / the proto-taxonomy defeater).
- `gold_action.posture_basis` — **standard-derived**: block-vs-proceed follows
  ASME V&V 40 (credibility bounded to the validated domain), so *proceeding when
  the standard says block* is the dangerous error regardless of taste.
- `gold_action.selection_basis` — **`provisional-self-adjudicated`**: the specific
  §5B class among coherent ones is the row-builder's reasoning, **not** independent.

## What this corpus is — and is not — ready for

- **Explanation gate: ready.** `harness/bakeoff/score.py` scores the gate on the
  **grounded posture** (dangerous-error + selective-coverage), not the
  self-adjudicated selection. `selection_accuracy` is reported but **provisional**.
- **Disposition gate: NOT ready.** Scoring a disposition gate against
  self-adjudicated gold is the tautology one level up (addendum). Before promotion
  the §5B selections need **expert/solver-derived re-adjudication**.
- **Size:** a couple dozen genuinely-hard cells is enough to *separate* models
  (addendum), but small-N reads stay a **preview** in `gate_read` — a clear routes
  to grow-and-rerun, never straight to a verdict.
