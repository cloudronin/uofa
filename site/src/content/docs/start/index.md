This section walks you from zero to a signed, validated UofA evidence package. Pick the path that fits your workflow.

| Path | When to use |
|---|---|
| [Install](/start/install/) | Pip + Codespace + Java prerequisites |
| [Your first UofA](/start/first-uofa/) | Scaffold a project, fill in JSON-LD, sign, validate |
| [Excel on-ramp](/start/from-excel/) | Fill an Excel workbook, import it, get a signed JSON-LD package in one command |

The Excel path is the fastest on-ramp for simulation engineers who do not work directly in JSON-LD. The first-UofA path is the canonical introduction to the UofA construct itself.

If you already have an existing v0.3 package and want to upgrade it to v0.5, see the migration command in the [CLI reference](/reference/cli/#migrate).
